# Eventide Project Context

## 🎯 Purpose
Eventide is a high-level Flutter plugin for native calendar access (iOS/Android) with a focus on ease-of-use and automatic permission handling. It is maintained by **SNCF Connect & Tech**.

## 🏗️ Architecture

### 1. Communication Layer (Pigeon)
- **Source of Truth**: `pigeons/calendar_api.dart`. All changes to the platform interface MUST start here.
- **Generated Files**:
  - Dart: `lib/src/calendar_api.g.dart`
  - Swift: `ios/eventide/Sources/eventide/CalendarApi.g.swift`
  - Kotlin: `android/src/main/kotlin/sncf/connect/tech/eventide/CalendarApi.g.kt`
- **Rule**: NEVER manually edit generated files. Run `dart run pigeon --input pigeons/calendar_api.dart` to update them.

### 2. Implementation Folders
- `lib/src/extensions/`: Contains domain-specific logic and UI-friendly extensions for the data models generated by Pigeon.
- `ios/eventide/Sources/`: Swift implementation using `EventKit`.
- `android/src/main/kotlin/`: Kotlin implementation using `CalendarProvider`.

### 3. Testing
- **Dart Tests**: `test/` (Unit tests for extensions and logic).
- **Native Tests**: `example/ios/EventideTests/` contains Swift unit tests for the native implementation.

## 🔑 Key Concepts & Invariants
- **Automatic Permissions**: Eventide handles its own permission flow. Check `doc/PLATFORM_SETUP.md` for required `Info.plist` and `AndroidManifest.xml` entries.
- **Privacy-First**: Methods like `createEventInDefaultCalendar` or `createEventThroughNativePlatform` are prioritized to minimize broad calendar access.
- **Data Models**: `Account`, `Calendar`, `Event`, `Attendee`, `Reminder` are the core entities.

## 🛠️ Dev Workflow
1. Modify `pigeons/calendar_api.dart`.
2. Generate glue code.
3. Implement in `ios/` (Swift) and `android/` (Kotlin).
4. Add extensions in `lib/src/extensions/` if needed.
5. Update `API_REFERENCE.md` for any public changes.
